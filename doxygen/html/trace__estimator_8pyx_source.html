58  """
59  Computes the trace of inverse of matrix based on stochastic Lanczos
60  quadrature (SLQ) method.
61 
62  :param A: invertible matrix or linear operator
63  :type A: numpy.ndarray, scipy.sparse matrix, or LinearOperator object
64 
65  :param max_num_samples: Number of Monte-Carlo trials
66  :type max_num_samples: unsigned int
67 
68  :param lanczos_degree: Lanczos degree
69  :type lanczos_degree: unsigned int
70 
71  :param gram: If ``True``, the Gram matrix ``A.T @ A`` is considered instead
72  of ``A`` itself. In this case, ``A`` itself can be a generic, but
73  square, matrix. If ``False``, matrix ``A`` is used, which then it has
74  to be symmetric and positive semi-definite.
75  :type gram: bool
76 
77  :param tolerance: The tolerance for the acceptable error of Lanczos
78  tri-diagonalization, or Golub-Kahn bi-diagonalization.
79  :type tolerance: DataType
80 
81  :return: Trace of ``A``
82  :rtype: float
83 
84  .. note::
85 
86  In Lanczos tri-diagonalization method, :math:`\\theta`` is the
87  eigenvalue of ``T``. However, in Golub-Kahn bi-diagonalization method,
88  :math:`\\theta` is the singular values of ``B``. The relation between
89  these two methods are as follows: ``B.T*B`` is the ``T`` for ``A.T*A``.
90  That is, if we have the input matrix ``A.T*T``, its Lanczos
91  tri-diagonalization ``T`` is the same matrix as if we bi-diagonalize
92  ``A`` (not ``A.T*A``) with Golub-Kahn to get ``B``, then ``T = B.T*B``.
93  This has not been highlighted in the above paper.
94 
95  To correctly implement Golub-Kahn, here :math:`\\theta` should be the
96  singular values of ``B``, **NOT** the square of the singular values of
97  ``B`` (as described in the above paper incorrectly!).
98 
99  Technical notes:
100  This function is a wrapper to a cython function. In cython we cannot
101  have function arguments with default values (neither ``cdef``,
102  ``cpdef``). As a work around, this function (defined with ``def``)
103  accepts default values for arguments, and then calls the cython
104  function (defined with ``cdef``).
105 
106  Reference:
107  * `Ubaru, S., Chen, J., and Saad, Y. (2017)
108  <https://www-users.cs.umn.edu/~saad/PDF/ys-2016-04.pdf>`_,
109  Fast Estimation of :math:`\\mathrm{tr}(F(A))` Via Stochastic Lanczos
110  Quadrature, SIAM J. Matrix Anal. Appl., 38(4), 1075-1099.
111  """
