24 def slq_method(
25  A,
26  gram=False,
27  p=1.0,
28  return_info=False,
29  parameters=None,
30  min_num_samples=10,
31  max_num_samples=50,
32  error_atol=None,
33  error_rtol=1e-2,
34  confidence_level=0.95,
35  outlier_significance_level=0.001,
36  lanczos_degree=20,
37  lanczos_tol=None,
38  orthogonalize=0,
39  num_threads=0,
40  num_gpu_devices=0,
41  gpu=False,
42  verbose=False,
43  plot=False):
44  """
45  Log-determinant of non-singular matrix or linear operator using stochastic
46  Lanczos quadrature method.
47 
48  Given the symmetric matrix or linear operator :math:`\\mathbf{A}` and the
49  real exponent :math:`p`, the following is computed:
50 
51  .. math::
52 
53  \\mathrm{logdet} \\left(\\mathbf{A}^p \\right) = p \\log_e \\vert
54  \\det (\\mathbf{A}) \\vert.
55 
56  If ``gram`` is `True`, then :math:`\\mathbf{A}` in the above is replaced by
57  the Gramian matrix :math:`\\mathbf{A}^{\\intercal} \\mathbf{A}`, and the
58  following is instead computed:
59 
60  .. math::
61 
62  \\mathrm{logdet} \\left((\\mathbf{A}^{\\intercal}\\mathbf{A})^p
63  \\right) = 2p \\log_e \\vert \\det (\\mathbf{A}) \\vert.
64 
65  If :math:`\\mathbf{A} = \\mathbf{A}(t)` is a linear operator of the class
66  :class:`imate.AffineMatrixFunction` with the parameter :math:`t`, then for
67  an input tuple :math:`t = (t_1, \\dots, t_q)`, an array output of the
68  size :math:`q` is returned, namely:
69 
70  .. math::
71 
72  \\mathrm{logdet} \\left((\\mathbf{A}(t_i))^p \\right),
73  \\quad i=1, \\dots, q.
74 
75  Parameters
76  ----------
77 
78  A : numpy.ndarray, scipy.sparse, :class:`imate.Matrix`, or \
79  :class:`imate.AffineMatrixFunction`
80  A non-singular sparse or dense matrix or linear operator. If ``gram``
81  is `False`, then `A` should be symmetric.
82 
83  .. warning::
84 
85  The symmetry of `A` is not pre-checked by this function. If
86  ``gram`` is `False`, make sure `A` is symmetric.
87 
88  gram : bool, default=False
89  If `True`, the log-determinant of the Gramian matrix,
90  :math:`(\\mathbf{A}^{\\intercal}\\mathbf{A})^p`, is computed. The
91  Gramian matrix itself is not directly computed. If `False`, the
92  log-determinant of :math:`\\mathbf{A}^p` is computed.
93 
94  p : float, default=1.0
95  The real exponent :math:`p` in :math:`\\mathbf{A}^p`.
96 
97  return_info : bool, default=False
98  If `True`, this function also returns a dictionary containing
99  information about the inner computation, such as process time,
100  algorithm settings, etc.
101 
102  parameters : array_like [`float`], default=one
103  This argument is relevant if `A` is a type of
104  :class:`AffineMatrixFunction`. By this argument, multiple inquiries,
105  :math:`(t_1, \\dots, t_q)`, can be passed to the parameter :math:`t` of
106  the linear operator :math:`\\mathbf{A}(t)`. The output of this function
107  becomes an array of the size :math:`q` corresponding to each of the
108  input matrices :math:`\\mathbf{A}(t_i)`.
109 
110  min_num_samples : int, default=10
111  The minimum number of Monte-Carlo samples. If the convergence criterion
112  is reached before finishing the minimum number of iterations, the
113  iterations are forced to continue till the minimum number of iterations
114  is finished. This value should be smaller than
115  ``maximum_num_samples``.
116 
117  max_num_samples : int, default=50
118  The maximum number of Monte-Carlo samples. If the convergence criterion
119  is not reached by the maximum number of iterations, the iterations are
120  forced to stop. This value should be larger than
121  ``minimum_num_samples``.
122 
123  error_atol : float, default=None
124  Tolerance of the absolute error of convergence of the output. Once the
125  error of convergence reaches ``error_atol + error_rtol * output``, the
126  iteration is terminated. If the convergence criterion is not met by the
127  tolerance, then the iterations continue till reaching
128  ``max_num_samples`` iterations. If `None`, the termination criterion
129  does not depend on this parameter.
130 
131  error_rtol : float, default=None
132  Tolerance of the relative error of convergence of the output. Once the
133  error of convergence reaches ``error_atol + error_rtol * output``, the
134  iteration is terminated. If the convergence criterion is not met by the
135  tolerance, then the iterations continue till reaching
136  ``max_num_samples`` iterations. If `None`, the termination criterion
137  does not depend on this parameter.
138 
139  confidence_level : float, default=0.95
140  Confidence level of error, which is a number between `0` and `1`. The
141  error of convergence of the population of samples is defined by their
142  standard deviation times the Z-score, which depends on the confidence
143  level. See notes below for details.
144 
145  outlier_significance_level : float, default=0.001
146  One minus the confidence level of the uncertainty of the outliers of
147  the output samples. This is a number between `0` and `1`.
148 
149  lanczos_degree : int, default=20
150  The number of Lanczos recursive iterations. The larger Lanczos degree
151  leads to better estimation. The computational cost quadratically
152  increases with the Lanczos degree.
153 
154  lanczos_tol : float, default=None
155  The tolerance to stop the Lanczos recursive iterations before
156  the end of iterations reached. If the tolerance is not met, all the
157  iterations (total of ``lanczos_degree`` iterations) continue till the
158  end. If set to `None` (default value), the machine' epsilon precision
159  is used. The machine's epsilon precision is as follows:
160 
161  * For 32-bit, machine precision is
162  :math:`2^{-23} = 1.1920929 \\times 10^{-7}`.
163  * For 64-bit, machine precision is
164  :math:`2^{-52} = 2.220446049250313 \\times 10^{-16}`,
165  * For 128-bit, machine precision is
166  :math:`2^{-63} = -1.084202172485504434 \\times 10^{-19}`.
167 
168  orthogonalize : int, default=0
169  Indicates whether to re-orthogonalize the eigenvectors during Lanczos
170  recursive iterations.
171 
172  * If set to `0`, no orthogonalization is performed.
173  * If set to a negative integer or an integer larger than
174  ``lanczos_degree``, a newly computed eigenvector is orthogonalized
175  against all the previous eigenvectors (also known as
176  `full reorthogonalization`).
177  * If set to a positive integer, say `q`, but less than
178  ``lanczos_degree``, the newly computed eigenvector is
179  orthogonalized against a window of last `q` previous eigenvectors
180  (known as `partial reorthogonalization`).
181 
182  num_threads : int, default=0
183  Number of processor threads to employ for parallel computation on CPU.
184  If set to `0` or a number larger than the available number of threads,
185  all threads of the processor are used. The parallelization is performed
186  over the Monte-Carlo iterations.
187 
188  num_gpu_devices : int default=0
189  Number of GPU devices (if available) to use for parallel multi-GPU
190  processing. If set to `0`, the maximum number of available GPU devices
191  is used. This parameter is relevant if ``gpu`` is `True`.
192 
193  gpu : bool, default=False
194  If `True`, the computations are performed on GPU devices where the
195  number of devices can be set by ``num_gpu_devices``. If no GPU device
196  is found, it raises an error.
197 
198  .. note::
199  When performing `repetitive` computation on the same matrix on GPU,
200  it is recommended to input `A` as an instance of
201  :class:`imate.Matrix` class instead of `numpy` or `scipy` matrices.
202  See examples below for clarification.
203 
204  verbose : bool, default=False
205  Prints extra information about the computations.
206 
207  plot : bool, default=False
208  Plots convergence of samples. For this, the packages `matplotlib` and
209  `seaborn` should be installed. If no display is available (such as
210  running this code on remote machines), the plots are saved as an `SVG`
211  file in the current directory.

219  then the output is an array.

334  Notes
335  -----
336 
337  **Computational Complexity:**
338 
339  This method uses stochastic Lanczos quadrature (SLQ), which is a randomized
340  algorithm. The computational complexity of this method is
341 
342  .. math::
343 
344  \\mathcal{O}((\\rho n^2 + n l) s l),
345 
346  where :math:`n` is the matrix size, :math:`\\rho` is the density of
347  sparse matrix (for dense matrix, :math:`\\rho=1`), :math:`l` is the
348  Lanczos degree (set with ``lanczos_degree``), and :math:`s` is the number
349  of samples (set with ``min_num_samples`` and ``max_num_samples``).
350 
351  This method can be used on very large matrices (:math:`n > 2^{12}`). The
352  solution is an approximation.
353 
354  **Input Matrix:**
355 
356  The input `A` can be either of:
357 
358  * A matrix, such as `numpy.ndarray`, or `scipy.sparse`.
359  * A linear operator representing a matrix using :class:`imate.Matrix`.
360  * A linear operator representing a one-parameter family of an affine matrix
361  function :math:`t \\mapsto \\mathbf{A} + t\\mathbf{B}` using
362  :class:`imate.AffineMatrixFunction`.
363 
364  **Output:**
365 
366  The output is a scalar. However, if `A` is the linear operator
367  :math:`\\mathbf{A}(t) = \\mathbf{A} + t \\mathbf{B}` where :math:`t` is
368  given as the tuple :math:`t = (t_1, \\dots, t_q)`, then the output of this
369  function is an array of size :math:`q` corresponding to the
370  log-determinant of each :math:`\\mathbf{A}(t_i)`.
371 
372  .. note::
373 
374  When `A` represents
375  :math:`\\mathbf{A}(t) = \\mathbf{A} + t \\mathbf{I}`, where
376  :math:`\\mathbf{I}` is the identity matrix, and :math:`t` is given by
377  a tuple :math:`t = (t_1, \\dots, t_q)`, the computational cost of an
378  array output of size `q` is the same as computing for a single
379  :math:`t_i`. Namely, the log-determinant of only
380  :math:`\\mathbf{A}(t_1)` is computed, and the log-determinant of the
381  rest of :math:`i=2, \\dots, q` are obtained from the result of
382  :math:`t_1` immediately.
383 
384  **Algorithm:**
385 
386  If ``gram`` is `False`, the Lanczos tri-diagonalization method is
387  used. This method requires only matrix-vector multiplication. If
388  ``gram`` is `True`, the Golub-Kahn bi-diagonalization method is used. This
389  method requires both matrix-vector multiplication and transposed-matrix
390  vector multiplications.
391 
392  **Convergence criterion:**
393 
394  Let :math:`n_{\\min}` and :math:`n_{\\max}` be the minimum and maximum
395  number of iterations respectively defined by ``min_num_samples`` and
396  ``max_num_samples``. The iterations terminate at
397  :math:`n_{\\min} \\leq i \\leq n_{\\max}` where :math:`i` is the
398  iteration counter. The iterations stop earlier at :math:`i < n_{\\max}` if
399  the convergence error of the mean of the samples is satisfied, as follows.
400 
401  Suppose :math:`s(j)` and :math:`\\sigma(i)` are respectively the mean and
402  standard deviation of samples after :math:`j` iterations. The error of
403  convergence, :math:`e(j)`, is defined by
404 
405  .. math::
406 
407  e(j) = \\frac{\\sigma(j)}{\\sqrt{j}} Z
408 
409  where :math:`Z` is the Z-score defined by
410 
411  .. math::
412 
413  Z = \\sqrt{2} \\mathrm{erf}^{-1}(\\phi).
414 
415  In the above, :math:`\\phi` is the confidence level and set by
416  ``confidence_level`` argument, and :math:`\\mathrm{erf}^{-1}` is the
417  inverse error function. A confidence level of 95%, for instance, means that
418  the Z-score is 1.96, which means the confidence interval is
419  :math:`\\pm 1.96 \\sigma`.
420 
421  The termination criterion is
422 
423  .. math::
424 
425  e(j) < \\epsilon_a + s(j) \\epsilon_r,
426 
427  where :math:`\\epsilon_{a}` and :math:`\\epsilon_r` are the absolute and
428  relative error tolerances respectively, and they are set by ``error_atol``
429  and ``error_rtol``.
430 
431  **Convergence for the case of multiple parameters:**
432 
433  When `A` is a type of :class:`imate.AffineMatrixFunction` representing the
434  affine matrix function :math:`\\mathbf{A}(t) = \\mathbf{A} + t \\mathbf{B}`
435  and if multiple parameters :math:`t_i`, :math:`i=1,\\dots, q` are passed to
436  this function through ``parameters`` argument, the convergence criterion
437  has to be satisfied for each of :math:`\\mathbf{A}(t_i)`. Specifically, the
438  iterations are terminated as follows:
439 
440  * If :math:`\\mathbf{B}` is the identity matrix, iterations for all
441  :math:`\\mathbf{A}(t_i)` continue till the convergence criterion for
442  *all* :math:`t_i` are satisfied. That is, even if :math:`t=t_i` is
443  converged but :math:`t=t_j` has not converged yet, the iterations for
444  :math:`t=t_i` will continue.
445  :
446  * If :math:`\\mathbf{B}` is not the identity matrix, the iterations for
447  each of :math:`t_i` are independent. That is, if :math:`t=t_i` converges,
448  the iterations for that parameter will stop regardless of the convergence
449  status of other parameters.
450

483  Examples
484  --------
485 
486  **Symmetric Input Matrix:**
487 
488  The `slq` method requires the input matrix of :func:`imate.logdet` function
489  to be symmetric when ``gram`` is `False`. For the first example, generate a
490  symmetric sample matrix using :func:`imate.toeplitz` function as follows:
491 
492  .. code-block:: python
493 
494  >>> # Import packages
495  >>> from imate import toeplitz
496 
497  >>> # Generate a symmetric matrix by setting gram=True.
498  >>> A = toeplitz(2, 1, size=100, gram=True)
499 
500  In the above, by passing ``gram=True`` to :func:`imate.toeplitz` function,
501  the Gramian of the Toeplitz matrix is returned, which is symmetric. Compute
502  the log-determinant of :math:`\\mathbf{A}^{2.5}`:
503 
504  .. code-block:: python
505 
506  >>> # Import packages
507  >>> from imate import logdet
508 
509  >>> # Compute the log-determinant
510  >>> logdet(A, gram=False, p=2.5, method='slq')
511  277.00578301560057
512 
513  .. note:
514 
515  Since `slq` is a stochastic method, the above result slightly
516  differs after each run.
517 
518  **Gramian Matrix:**
519 
520  Passing ``gram=True`` to :func:`imate.logdet` function uses the Gramian
521  of the input matrix. n this case, the input matrix can be non-symmetric.
522  In the next example, generate a non-symmetric matrix, :math:`\\mathbf{B}`,
523  then compute the log-determinant of
524  :math:`(\\mathbf{B}^{\\intercal} \\mathbf{B})^{2.5}`.
525 
526  .. code-block:: python
527 
528  >>> # Generate a non-symmetric matrix by setting gram=False
529  >>> B = toeplitz(2, 1, size=100, gram=False)
530 
531  >>> # Compute log-determinant of Gramian by passing gram=True
532  >>> logdet(B, gram=True, p=2.5, method='slq')
533  275.15518680196476
534 
535  Note that the result of the two examples in the above are the similar since
536  the input matrix :math:`\\mathbf{A}` of the first example is the Gramian of
537  the input matrix :math:`\\mathbf{B}` in the second example, that is
538  :math:`\\mathbf{A} = \\mathbf{B}^{\\intercal} \\mathbf{B}`. Since the
539  second example uses the Gramian of the input matrix, it computes the same
540  quantity as the first example.

689  **Matrix operator:**
690 
691  Use an object of :class:`imate.Matrix` class as an alternative method to
692  pass the matrix `A` to the `logdet` function.
693 
694  .. code-block:: python
695  :emphasize-lines: 8
696 
697  >>> # Import matrix operator
698  >>> from imate import toeplitz, logdet, Matrix
699 
700  >>> # Generate a sample matrix (a toeplitz matrix)
701  >>> A = toeplitz(2, 1, size=100, gram=True)
702 
703  >>> # Create a matrix operator object from matrix A
704  >>> Aop = Matrix(A)
705 
706  >>> # Compute log-determinant of Aop
707  >>> logdet(Aop, method='slq')
708  141.52929878934194
709 
710  An advantage of passing `Aop` (instead of `A`) to the `logdet` function
711  will be clear when using GPU.
712 
713  **Computation on GPU:**
714 
715  The argument ``gpu=True`` performs the computations on GPU. The following
716  example uses the object `Aop` created earlier.
717 
718  .. code-block:: python
719 
720  >>> # Compute log-determinant of Aop
721  >>> logdet(Aop, method='slq', gpu=True)
722  141.52929878934194
723 
724  The above function call triggers the object `Aop` to automatically load the
725  matrix data on the GPU.
726 
727  One could have used `A` instead of `Aop` in the above. However, an
728  advantage of using `Aop` (instead of the matrix `A` directly) is that by
729  calling the above `logdet` function (or another function) again on this
730  matrix, the data of this matrix does not have to be re-allocated on the GPU
731  device again. To highlight this point, call the above function again, but
732  this time, set ``gram`` to `True` to compute something different.
733 
734  .. code-block:: python
735 
736  >>> # Compute log-determinant of Aop
737  >>> logdet(Aop, method='slq', gpu=True, gram=True)
738  141.52929878934194
739 
740  In the above example, no data is needed to be transferred from CPU host to
741  GPU device again. However, if `A` was used instead of `Aop`, the data would
742  have been transferred from CPU to GPU again for the second time. The `Aop`
743  object holds the data on GPU for later use as long as this object does no
744  go out of the scope of the python environment. Once the variable `Aop` goes
745  out of scope, the matrix data on all the GPU devices will be cleaned
746  automatically.
