24 def traceinv(
25  A,
26  gram=False,
27  p=1,
28  return_info=False,
29  method='cholesky',
30  **options):
31  """
32  Trace of inverse of non-singular matrix or linear operator.
33 
34  Given the matrix or the linear operator :math:`\\mathbf{A}` and the real
35  exponent :math:`p`, the following is computed:
36 
37  .. math::
38 
39  \\mathrm{trace} \\left(\\mathbf{A}^{-p} \\right).
40 
41  If ``gram`` is `True`, then :math:`\\mathbf{A}` in the above is replaced by
42  the Gramian matrix :math:`\\mathbf{A}^{\\intercal} \\mathbf{A}`, and the
43  following is instead computed:
44 
45  .. math::
46 
47  \\mathrm{trace} \\left((\\mathbf{A}^{\\intercal}\\mathbf{A})^{-p}
48  \\right).
49 
50  If :math:`\\mathbf{A} = \\mathbf{A}(t)` is a linear operator of the class
51  :class:`imate.AffineMatrixFunction` with the parameter :math:`t`, then for
52  an input tuple :math:`t = (t_1, \\dots, t_q)`, an array output of the size
53  :math:`q` is returned, namely:
54 
55  .. math::
56 
57  \\mathrm{trace} \\left((\\mathbf{A}(t_i))^{-p} \\right),
58  \\quad i=1, \\dots, q.
59 
60  Parameters
61  ----------
62 
63  A : numpy.ndarray, scipy.sparse, :class:`imate.Matrix`, or \
64  :class:`imate.AffineMatrixFunction`
65  A non-singular sparse or dense matrix or linear operator. The linear
66  operators :class:`imate.Matrix` and :class:`imate.AffineMatrixFunction`
67  can be used only if ``method=slq``. See details in
68  :ref:`slq method <imate.traceinv.slq>`. If ``method=cholesky``, the
69  matrix `A` should be positive-definite. If ``method=slq`` and
70  ``gram=False``, the input matrix `A` should be symmetric.
71 
72  gram : bool, default=False
73  If `True`, the trace of the Gramian matrix,
74  :math:`(\\mathbf{A}^{\\intercal}\\mathbf{A})^{-p}`, is computed. The
75  Gramian matrix itself is not directly computed. If `False`, the
76  trace of :math:`\\mathbf{A}^{-p}` is computed.
77 
78  p : float, default=1.0
79  The exponent :math:`p` in :math:`\\mathbf{A}^{-p}`.
80 
81  * If ``method=eigenvalue``, :math:`p` can be any real number.
82  * If ``method=cholesky``, :math:`p` should be an integer.
83  * If ``method=hutchinson``, :math:`p` should be an integer.
84  * If ``method=slq``, :math:`p` should be non-negative real number.
85 
86  .. note::
87 
88  If :math:`p < 0`, use :func:`imate.trace` function.
89 
90  return_info : bool, default=False
91  If `True`, this function also returns a dictionary containing
92  information about the inner computation, such as process time,
93  algorithm settings, etc. See the documentation for each `method` for
94  details.
95 
96  method : {'eigenvalue', 'cholesky', 'hutchinson', 'slq'}, \
97  default='cholesky'
98  The method of computing trace. See documentation for each
99  method:
100 
101  * :ref:`eigenvalue <imate.traceinv.eigenvalue>`
102  * :ref:`cholesky <imate.traceinv.cholesky>`
103  * :ref:`hutchinson <imate.traceinv.hutchinson>`
104  * :ref:`slq <imate.traceinv.slq>`
105 
106  options : `**kwargs`
107  Extra arguments that are specific to each method. See the documentation
108  for each `method` for details.
109 
110  Returns
111  -------
112 
113  traceinv : float or numpy.array
114  Trace of inverse of matrix. If ``method=slq`` and if `A` is of type
115  :class:`imate.AffineMatrixFunction` with an array of ``parameters``,
116  then the output is an array.
117 
118  info : dict
119  (Only if ``return_info`` is `True`) A dictionary of information with at
120  least the following keys. Further keys specific to each method can be
121  found in the documentation of each method.
122 
123  * ``matrix``:
124  * ``data_type``: `str`, {`float32`, `float64`, `float128`}, type of
125  the matrix data.
126  * ``gram``: `bool`, whether the matrix `A` or its Gramian is
127  considered.
128  * ``exponent``: `float`, the exponent `p` in
129  :math:`\\mathbf{A}^{-p}`.
130  * ``size``: `int`, The size of matrix `A`.
131  * ``sparse``: `bool`, whether the matrix `A` is sparse or dense.
132  * ``nnz``: `int`, if `A` is sparse, the number of non-zero elements
133  of `A`.
134  * ``density``: `float`, if `A` is sparse, the density of `A`, which
135  is the `nnz` divided by size squared.
136  * ``num_inquiries``: `int`, The size of inquiries of each parameter
137  of the linear operator `A`. If `A` is a matrix, this is always
138  `1`. For more details see :ref:`slq method <imate.traceinv.slq>`.
139 
140  * ``device``:
141  * ``num_cpu_threads``: `int`, number of CPU threads used in shared
142  memory parallel processing.
143  * ``num_gpu_devices``: `int`, number of GPU devices used in the
144  multi-GPU (GPU farm) computation.
145  * ``num_gpu_multiprocessors``: `int`, number of GPU
146  multi-processors.
147  * ``num_gpu_threads_per_multiprocessor``: `int`, number of GPU
148  threads on each GPU multi-processor.
149 
150  * ``time``:
151  * ``tot_wall_time``: `float`, total elapsed time of computation.
152  * ``alg_wall_time``: `float`, elapsed time of computation during
153  only the algorithm execution.
154  * ``cpu_proc_time``: `float`, CPU processing time of computation.
155 
156  * ``solver``:
157  * ``version``: `str`, version of imate.
158  * ``method``: `str`, method of computation.
159 
160  Raises
161  ------
162 
163  LinAlgError
164  If ``method=cholesky`` and `A` is not positive-definite.
165 
166  ImportError
167  If the package has not been compiled with GPU support, but ``gpu`` is
168  `True`. Either set ``gpu`` to `False` to use the existing installed
169  package. Alternatively, export the environment variable ``USE_CUDA=1``
170  and recompile the source code of the package.

179  Notes
180  -----
181 
182  **Method of Computation:**
183 
184  See documentation for each method below.
185 
186  * :ref:`eigenvalue <imate.traceinv.eigenvalue>`: uses spectral decomposition.
187  Suitable for small matrices (:math:`n < 2^{12}`). The solution is exact.
188  * :ref:`cholesky <imate.traceinv.cholesky>`: uses Cholesky decomposition.
189  Suitable for moderate-size matrices (:math:`n < 2^{15}`). Can only be
190  applied to positive-definite matrices. The solution is exact.
191  * :ref:`hutchinson <imate.traceinv.hutchinson>`: uses stochastic Hutchinson
192  method, which is a randomized algorithm. Can be used on very large
193  matrices (:math:`n > 2^{12}`). The solution is an approximation.
194  * :ref:`slq <imate.traceinv.slq>`: uses stochastic Lanczos quadrature
195  (SLQ), which is a randomized algorithm. Can be used on very large
196  matrices (:math:`n > 2^{12}`). The solution is an approximation.
197 
198  **Input Matrix:**
199 
200  The input `A` can be either of:
201 
202  * A matrix, such as `numpy.ndarray`, or `scipy.sparse`.
203  * A linear operator representing a matrix using :class:`imate.Matrix` (
204  only if ``method=slq``).
205  * A linear operator representing a one-parameter family of an affine matrix
206  function :math:`t \\mapsto \\mathbf{A} + t\\mathbf{B}`, using
207  :class:`imate.AffineMatrixFunction` (only if ``method=slq``).
208 
209  **Output:**
210 
211  The output is a scalar. However, if `A` is the linear operator of the type
212  :class:`imate.AffineMatrixFunction` representing the matrix function
213  :math:`\\mathbf{A}(t) = \\mathbf{A} + t \\mathbf{B}`, then if the parameter
214  :math:`t` is given as the tuple :math:`t = (t_1, \\dots, t_q)`, then the
215  output of this function is an array of size :math:`q` corresponding to the
216  trace of each :math:`\\mathbf{A}(t_i)`.
217 
218  .. note::
219 
220  When `A` represents
221  :math:`\\mathbf{A}(t) = \\mathbf{A} + t \\mathbf{I}`, where
222  :math:`\\mathbf{I}` is the identity matrix, and :math:`t` is given by
223  a tuple :math:`t = (t_1, \\dots, t_q)`, by setting ``method=slq``, the
224  computational cost of an array output of size `q` is the same as
225  computing for a single :math:`t_i`. Namely, the trace of inverse of
226  only :math:`\\mathbf{A}(t_1)` is computed, and the trace of inverse of
227  the rest of :math:`q=2, \\dots, q` are obtained from the result of
228  :math:`t_1` immediately.

233  **Sparse matrix:**
234 
235  Compute the trace of inverse of a sample sparse Toeplitz matrix created by
236  :func:`imate.toeplitz` function.
237 
238  .. code-block:: python
239 
240  >>> # Import packages
241  >>> from imate import toeplitz, traceinv
242 
243  >>> # Generate a sample matrix (a toeplitz matrix)
244  >>> A = toeplitz(2, 1, size=100)
245 
246  >>> # Compute trace of inverse with Cholesky method (default method)
247  >>> traceinv(A)

302  **Large matrix:**
303 
304  Compute trace of inverse of a very large sparse matrix using `SLQ` method.
305  This method does not compute the trace of inverse exactly, rather, the
306  result is an approximation using Monte-Carlo sampling. The following
307  example uses at least `100` samples.
308 
309  .. code-block:: python
310 
311  >>> # Generate a matrix of size one million
312  >>> A = toeplitz(2, 1, size=1000000, gram=True)
313 
314  >>> # Approximate of trace of inverse using stochastic Lanczos
315  >>> # quadrature with at least 100 Monte-Carlo sampling
316  >>> ti, info = traceinv(A, method='slq', min_num_samples=100,
317  ... max_num_samples=200, return_info=True)
318  >>> print(ti)
319  333440.32441422355
320 
321  >>> # Find the time it took to compute the above
322  >>> print(info['time'])
323  {
324  'tot_wall_time': 16.256937585072592,
325  'alg_wall_time': 16.236278533935547,
326  'cpu_proc_time': 118.06356006200001
327  }
328 
329  Compare the result of the above approximation with the exact solution of
330  the trace of inverse using the analytic relation for Toeplitz matrix. See
331  :func:`imate.sample_matrices.toeplitz_traceinv` for details.
332 
333  .. code-block:: python
334 
335  >>> from imate.sample_matrices import toeplitz_traceinv
336  >>> toeplitz_traceinv(2, 1, size=1000000, gram=True)
337  333333.2222222222
338 
339  It can be seen that the error of approximation is :math:`0.032 \\%`. This
340  accuracy is remarkable considering that the computation on such a large
341  matrix took only a 16 seconds. Computing the trace of inverse of such a
342  large matrix using any of the exact methods (such as ``cholesky`` or
343  ``eigenvalue``) is infeasible.
344 
345  Alternatively, for large matrices, the Hutchinson method can be used:
346 
347  .. code-block:: python
348 
349  >>> # Approximate of trace of inverse using hutchinson method
350  >>> # with at least 100 Monte-Carlo sampling
351  >>> ti, info = traceinv(A, method='hutchinson', min_num_samples=100,
352  ... max_num_samples=200, return_info=True)
353  >>> print(ti)
354  333315.65986928536
355 
356  >>> # Find the time it took to compute the above
357  >>> print(info['time'])
358  {
359  'tot_wall_time': 275.63905002200045,
360  'alg_wall_time': 215.36715987394564,
361  'cpu_proc_time': 860.377073873
362  }
363 
364  The above result with Hutchinson's method is remarkably close to the true
365  value with only `0.005 \\%` error, however, it takes longer time compared
366  to the SLQ method.
